零成本搭建：用 OpenClaw 将 GPT 大模型免费接入 QQ/微信/Telegram/飞书

发表于2026-05-24|更新于2026-07-23|toolssoftwareaichatgptopenclaw
|阅读量:

想要让 AI 机器人直接在你的常用聊天软件里打工，实现自动回复、智能助理或自动化日常事务，但又不想折腾复杂的 API Key 或者承担高昂的 Token 费用？

开源神器 OpenClaw 正好能完美解决这个痛点。它支持将强大的 GPT 模型免 API 密钥直接对接至 QQ、Telegram、飞书以及 macOS 端的个人微信。本篇教程将手把手带你完成全平台低门槛部署，打造属于你的全能私有 AI 助手。

二、 核心通道对接实操

OpenClaw 提供了强大的多渠道（Channels）支持，你可以根据需求选择对接以下平台：

2. QQ 机器人对接

  1. 确保电脑已安装并登录了最新版的 QQ 官方客户端。
  2. 浏览器访问 QQ 机器人开放平台 并登录你的账号。
  3. 点击右上角的 “创建机器人”，系统会自动为你分配专属的对接参数，并直接生成一条一键对接命令。
  4. 将该命令复制到终端运行，对接成功后即可在 QQ 中通过机器人直接操控电脑或进行 AI 对话。

3. Telegram 机器人对接

  1. 在 Telegram 中搜索官方机器人之父：@BotFather。

  2. 向其发送 /start，接着发送 /newbot 开启创建流程，按照提示设定机器人的名称和用户名（Username）。

  3. 创建完成后，复制 BotFather 给你生成的 Bot Token 并妥善保存。

4. 飞书（Lark）企业自建应用对接

飞书的结构相对严谨，需要利用长连接（WebSocket）模式引入：

第一步：创建应用 访问 飞书开放平台，点击 “创建企业自建应用”。填入应用名称与图标后，进入凭证页面，复制并保存你的 App ID 和 App Secret。

第二步：终端配置通道 在终端输入 openclaw channels add，在菜单中选择 Feishu/Lark (飞书)，随后依次填入对应的 App Secret 和 App ID。连接模式选择国内域名，群聊策略建议选择 Open（响应所有群聊）。

第三步：开通机器人能力与权限 回到飞书网页端后台，在左侧菜单进入 “添加应用能力” -> “机器人” 点击添加。接着进入 “权限管理”，选择批量导入，将官方提供的标准 JSON 权限表粘贴进去，确保涵盖消息读写（im:message）等核心权限。

第四步：事件订阅与发布 在 “事件与回调” 中，将订阅方式改为 长连接接收事件（WebSocket）。随后添加 im.message.receive_v1（接收消息）等基础事件。最后在 “版本管理与发布” 中创建应用版本并提交企业内发布。

添加以下事件：

接收消息

im.message.receive_v1

消息已读回执

im.message.message_read_v1

机器人进群

im.chat.member.bot.added_v1

机器人被移出群

im.chat.member.bot.deleted_v1

发布应用

总结

OpenClaw 的硬核之处在于其高集成度与零成本限制。通过一套本地网关，就能同时给 QQ、Telegram、飞书等多个生产力与社交工具注入大模型的核心能力，非常适合用来搭建个人专属的轻量化自动化工作流。
